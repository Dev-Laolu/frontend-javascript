## üß© Project Highlights

### **Task 0 ‚Äì Interface for a Student**
Create a simple interface for a student and render student data in an HTML table using TypeScript and DOM manipulation.

### **Task 1 ‚Äì Teacher Interface**
Define a `Teacher` interface with modifiable and optional fields. Demonstrates flexibility in object structure and optional properties.

### **Task 2 ‚Äì Directors Extension**
Extend the `Teacher` interface to include additional attributes (`numberOfReports`), exploring inheritance in interfaces.

### **Task 3 ‚Äì PrintTeacher Function**
Implement a function interface and a print function that formats a teacher‚Äôs name as `J. Doe`.

### **Task 4 ‚Äì Student Class**
Define a `StudentClass` with methods `displayName()` and `workOnHomework()`. Introduces class structure and interface-based typing.

### **Task 5 ‚Äì Advanced Types (Part 1)**
Implement multiple interfaces (`TeacherInterface`, `DirectorInterface`) and classes with different behaviors.  
Create a factory function to return specific class instances dynamically.

### **Task 6 ‚Äì Function Predicates**
Write functions that differentiate between types (`isDirector`) and perform specific actions based on employee roles.

### **Task 7 ‚Äì String Literal Types**
Demonstrate strict type enforcement by allowing only predefined string values (`"Math"` | `"History"`).

### **Task 8 ‚Äì Ambient Namespaces**
Define type declarations for third-party libraries and use ambient files (`.d.ts`) to describe non-TypeScript modules.

### **Task 9 ‚Äì Namespaces & Declaration Merging**
Work with **TypeScript namespaces** and explore how multiple declarations merge into a single structure.

### **Task 10 ‚Äì Subject Management**
Create subject classes (`Cpp`, `Java`, `React`) using merged declarations and polymorphic methods.

### **Task 11 ‚Äì Nominal Typing**
Introduce **branding** in TypeScript to differentiate types that share similar structure but represent different concepts (e.g., `MajorCredits` and `MinorCredits`).
